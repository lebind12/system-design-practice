## 페이지 속성 규약

| 속성 | 타입 | 값 규칙 |
|---|---|---|
| 제목 | title | **반드시 `[가상 면접 사례로 배우는 대규모 시스템 설계 기초] <챕터 주제>`** 형식. 예: `[가상 면접 사례로 배우는 대규모 시스템 설계 기초] URL 단축기` |
| 날짜 | date | 세션 진행 일자 (오늘) |
| 주제 | text | 한 문장 요약. 예: `분산 환경에서 초당 1000 QPS를 처리하는 URL 단축 서비스 구현` |
| 학습모드 | select | `대화형학습` |
| 학습내용 | text | 2~3 문장. "무엇을 만들었고 어떤 확장 실험을 했는가" |
| 핵심개념 | text | 쉼표 구분 키워드 (예: `base62, 해시충돌, 캐시, 301 vs 302`) |
| 태그 | multi_select | 아래 6개 중 해당하는 것만 선택: `캐시`, `로드밸런싱`, `데이터베이스`, `메시지큐`, `CDN`, `API설계` |

> **주의**: 태그 옵션은 DB에 이미 등록된 6개 외에는 사용하지 않습니다. 새 태그가 필요하면 사용자에게 먼저 물어봐 DB에 추가할지 확인합니다.

## 생성 절차 (Claude가 따를 순서)

1. 사용자에게 노션 기록 진행 동의 확인
2. 챕터 `README.md` 내용을 위 섹션 구조로 요약
3. `notion-create-pages` 호출 — parent 는 위 data source ID, properties 는 위 표대로
4. 생성된 페이지 URL 을 사용자에게 보여주고, 루트 `README.md` 인덱스의 "노션" 컬럼에 링크 추가
