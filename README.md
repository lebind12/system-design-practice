# System Design Practice

『**가상 면접 사례로 배우는 대규모 시스템 설계 기초**』(알렉스 쉬) 의 각 챕터를 **직접 구현**으로 학습하는 포트폴리오 저장소입니다. 책의 요약이 아니라, 책이 제시한 설계 문제를 실제로 돌려보고 벤치마크로 검증하며 마주친 의사결정을 기록하는 것이 목적입니다.

## Why this repo

책을 한 번 읽는 것과 직접 만들어 부하를 거는 것은 완전히 다른 경험입니다. 이 저장소는 그 간극을 좁히기 위해 챕터마다 다음을 수행합니다:

1. 요구사항과 규모를 수치로 먼저 정의
2. 최소 구동(MVP) → 확장 실험 2~3개를 단계적으로 적용
3. 각 단계에서 **k6 / wrk / hey 로 p50·p95·p99 지연시간과 RPS 측정**
4. 왜 A가 아닌 B를 선택했는지 **의사결정 트레이드오프**를 기록
5. 읽기만 했을 때는 보이지 않던 "배운 것"을 남김

구현 언어는 챕터마다 자유롭게 고릅니다 (Go, Python(FastAPI·Django·Flask), Java(Spring) 등) — 시스템 설계 사고를 언어에 종속시키지 않기 위함입니다.

## 저장소 구조

```
.
├── CLAUDE.md                 세션 공통 규약 (Claude Code 자동 로드)
├── docs/
│   ├── CHAPTER_WORKFLOW.md   9단계 챕터 진행 규약
│   └── NOTION_TEMPLATE.md    노션 기록 규약
├── _template/                새 챕터 스켈레톤 (cp -r 로 복사해 사용)
│   ├── README.md
│   ├── Makefile
│   ├── docker-compose.yml
│   └── .env.example
└── <chapter-name>/           각 챕터는 독립된 디렉토리 (인프라·앱·벤치 자체 소유)
    ├── README.md             요구사항·설계·벤치·회고
    ├── Makefile              up / down / run / test / bench / clean
    ├── docker-compose.yml    name 필드 = 디렉토리명 (Docker Desktop 그룹핑)
    ├── .env.example          커밋되는 환경 변수 구조
    └── .env                  실제 값 (커밋되지 않음)
```

각 챕터 디렉토리는 **독립적**으로 동작합니다. 하나의 챕터를 돌려보려면 그 디렉토리에 들어가 `make up && make run` 만 하면 됩니다.

## 읽는 순서 (면접관/리뷰어용)

1. 관심 있는 챕터 디렉토리의 `README.md` 열기
2. "요구사항 정의" → "개략적 규모 추정" → "상위 설계" 순서로 문제 정의 확인
3. "벤치마크 결과" 표에서 MVP 와 확장 실험 사이 수치 변화 비교
4. "의사결정과 트레이드오프" / "배운 것" 섹션이 이 저장소의 **실질적 콘텐츠**입니다

### 0번 — `scaling-foundations` 주의사항

책의 1·2·3장은 이론·메타 챕터이므로 **간략하게** 진행합니다. 구현 여지가 있는 부분(예: 1장의 단일 서버 → 수평 확장 단계별 구동, 2장의 규모 추정 계산기 스크립트)은 작은 lab 으로 만들고, 3장의 면접 공략법은 문서로만 정리합니다.

## 저작권 및 사용 범위

- 본 저장소는 책의 요약·번역·복제가 아니며, 독자의 **직접 구현 학습 기록**입니다.
- 책의 본문·도해는 포함되어 있지 않습니다. 각 챕터의 설계 문제는 책의 주제를 참조하되, 구현과 벤치·회고는 전적으로 저자의 작업입니다.
- 포트폴리오 목적의 개인 저장소이며 별도의 오픈소스 라이선스를 부여하지 않습니다.
